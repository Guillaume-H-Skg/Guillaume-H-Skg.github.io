Projet Tutoré S1 2021 - 2022

L'objectif de ce projet était de concevoir un site où un utilisateur peut s'inscrire/se connecter à leur compte afin de créer une partie pour inviter ses amis et jouer ensemble à un quizz de culture générale. Une partie administrateur est aussi implémentée, il peut ajouter, modifier et supprimer toutes les informations relatives aux utilisateurs et aux questions.

Lorsque l'utilisateur est connecté, il peut créer une partie. Il attribue une couleur à chaque joueur et saisit leurs pseudonymes à l'aide de l'autocompletion (les autres joueurs doivent aussi avoir un compte sur la plateforme). Des invitations sont donc envoyées aux joueurs par email. Une fois tous les joueurs présents la partie se lance. Le but est de choisir la difficulté de la question. Selon la difficulté de la question le joueur devra avancer ou reculer du nombre de cases correspondant à la difficulté de la question. Le jeu est donc tour par tour et pour gagner il faut obtenir 48 points (chaque question attribue un nombre de points selon la difficulté choisie).

IUT - Projet

Ce projet est un projet réalisé durant ma licence professionnelle, le principe était de découvrir Flutter.

Reminder - Flutter

Ce projet consistait à réaliser une application mobile avec Flutter et d'utiliser une base de données distante et de stocker des données en local. J'ai donc choisi de réaliser une application mobile permettant d'enregistrer des clients et de leur attribuer des rendez-vous. Il est aussi possible d'activer des notifications de rappel pour les différents rendez-vous de chaque client à l'heure désirée. Pour stocker les données distantes j'ai utilisé Firebase. Cela permet de garder une authentification persistante et il possible de se connecter avec plusieurs comptes pour gérer les différents utilisateurs. L'application fonctionne aussi en local avec le stockage du téléphone et ne nécessite aucune connexion.

IUT - Projet Tutore

Ce projet est un projet réalisé durant ma licence professionnelle, le but principal était de délivrer une application multisupport.

Projet Tutoré S2 - E-cabinet

L'application se présente sous la forme d'un site de gestion de cabinet d'avocat. Le principal objectif était de partir d'une seule base de code et de déployer l'application sur plusieurs supports, notamment le mobile avec Ionic et Electron pour l'application bureau. Le design n'était pas une priorité, l'application devait tout de même être présentable. Cette application utilise React, TypeScript et MUI pour le front-end. Une couche back-end en PHP a été réalisée pour les versions Ionic et Web. Pour la version Electron une base de données en local a été mise en place.
